17. Step 08 - Understanding the Need for Docker and Containers
------------------------------------------------------------------------
 1. Understanding the Need for containers  and Containers orchestration.
 2.The journey towards container and container orchestration started with microservices.
 
 Microservices:-
 ------------------
 1.Microservices is one of the most popular buzzwords of the last few years.
 2.Almost every enterprise out there is heading towards microservices architecture.
 
  Q)what is a microservices architecture?
  A)The fundamental concept behind microservice architecture is instead of building a large monolith application,how about building very, very small, independently deployable microservices?
 -- So microservices architecture is all about building small focused microservices, which are independently deployable.
 --The great thing about the microservices architecture is that it provides you with flexibility to innovate and build applications in different programming languages(go,java,python,JavaScript, etc)
 
 For example, 
 I can build movie service in :- go.
I can build customer service in :- Python.

I can build review service in :-Node.js, 
I can build booking service in :- Java.

However, as they say, with great flexibility comes great problems.
--One of the challenges with microservices architectures is the fact that deployments become very complex.

--How can we have one way of deploying and monitoring?Go Java, Python or JavaScript microservices?
   A) containers :-   Docker is one of the most popular ways of building container images
   2. Docker allows you create Docker images for each of these microservices,
   3)and you docker image containes all needs of microservice 
    1)it contains application run time(JDK/Python/NodeJs)
	2) it allows contains Application code and all the Dependencies that the application need to run.
	3)therefore you can run your Docker image the same way on any infrastructure because your Docker.
	--You can run it in your local machine, your corporate data center, or in the cloud the same way.
	--In summary, a Docker image contains everything that your microservice needs.


So what are the advantages of going for Docker?
A) 1.Docker is cloud neutral
  2. Docker also brings in standardizationSimplified operations - Consistent deployment,monitopring,logging
  3. Docker containers are lightweight.
4.The alternative to Docker containers is to make use of something called virtual machines.
5.Docker also provides very good isolation for your containers.
