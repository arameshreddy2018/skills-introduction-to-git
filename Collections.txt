Map Interface 
-----------------------------------------
1. interface Map<k,v>
2. interface sortedMap<k,v> extends Map
3.interface NavigableMap<k,v> extends sortedMap


Map Interface: 
1.This data structure stores data as key-value pairs, allowing users to associate a key (e.g., an element A) with a value (e.g., its count, 5). 
Core methods include put(key, value), get(key), remove(key), containsKey(), keySet(), and values(). 
Ordering of elements is not guaranteed in a standard map.

SortedMap Interface: 
1.This SortedMap extends the Map interface 
2.  but SortedMap ensures that elements are stored and retrieved in the order of their keys (either natural ordering or via a custom Comparator).
3.In sortedMap order of Elements is guaranteed order.
4. SortedMap woulb also provides some methods like subMap(K fromKey, K toKey), headMap(K toKey), tailMap(K fromKey), firstKey(), and lastKey() 

NavigableMap interface:-

The NavigableMap interface extends SortedMap 
Key Features of NavigableMap
------------------------------
Enhanced Navigation: 
1.It provides methods to find the closest matches for a given key, not just exact matches.
2.Descending Order Views: It allows traversing the map in both ascending (default) and descending key order.
NavigableMap also provide some methods:-
NavigableMap adds methods for retrieving entries based on their proximity to a given key, such as 
lowerKey()/lowerEntry(), floorKey()/floorEntry(), ceilingKey()/ceilingEntry(), and higherKey()/higherEntry().
3. It also includes methods to remove the first or last entry (pollFirstEntry()/pollLastEntry()) and
4. to obtain reverse order views of the map or its key set (descendingMap()/descendingKeySet()).
 You can find a complete list of methods in the official Java NavigableMap interface documentation.

 Blocking queue interface
 --------------------------------------
 the blocking queue interface is used when you want to supports operations that wait for the queue to become non-empty.
 
 usecase:-
 -----------
 I would want to get an element from the queue, but the queue does not have any elements right now in that kind of situation.

I want to wait for an element to become available in those kind of situations.We go for a blocking queue, so interface blocking queue supports those kind of use cases.
So basically I'm pulling a queue. The queue does not have any elements.I would want to still wait for some time and wait for an element to be available.

this Blocking queue interface also  have some methods
----------------------------------------------------------------
1.boolean add(E e) :-
-- Same as in Queue interface.
-- Inserts the specified element into queue IMMEDIATELY
-- Throws exception in case of failure.

2. boolean offer(E e):-
-- Same as in Queue interface.
-- Inserts the specified element into queue IMMEDIATELY
-- Return false in case of failure.

3.void put(E e):-
-- Inserts the specified element into this queue,waiting
-- if necessary for space to become availble.

4.boolean offer(E e, long timeout, TimeUnit unit) throws InterruptedException
--waiting upto the specifies wait time

5.E take() throws InterruptedException;
-- waits until element becomes available

6.E poll(long timeout,TimeUnit unit) throws InterruptedException;

comparator interface:-

 If you want to sort elements of a custom class in any way you want (custom order), you typically create a separate class that implements the Comparator interface. 
 the key difference in the compare() method is that it requires two objects as parameters to decide their order.

Comparable interface:-
--------------------------
1.If you want to define a natural ordering for a class, you must have that class implement the Comparable interface.
2.This "natural order" is the default way instances of that class will be sorted.
